# GHADS - دليل الموارد والخدمات المحلي وتنسيق المساعدات الإنسانية

## 📌 عن النظام (System Purpose)
منظومة **GHADS** هي تطبيق متكامل لإدارة وتنسيق المساعدات الإنسانية وتوزيع الموارد على العائلات النازحة والمؤسسات الشريكة. يهدف النظام إلى أتمتة العمليات الإغاثية وضمان التوزيع العادل للموارد لمنع الازدواجية في بيئات العمل الإنساني وأثناء فترات إعادة الإعمار الإقليمية.

## 🛠️ المشكلة التي يحلها النظام (The Problem It Solves)
في أوقات الأزمات، تواجه المؤسسات الإغاثية تحدياً كبيراً يتمثل في عشوائية التوزيع واستغلال الموارد، حيث قد تستلم بعض العائلات مساعدات متعددة في نفس اليوم من جهات مختلفة بينما تُحرم عائلات أخرى.
يقوم هذا النظام بحل المشكلة جذرياً عبر **آلية الفحص الذكي ومنع التكرار (Duplicate Aid Check)**، والتي تفحص سجلات قاعدة البيانات فورياً وتمنع الاستلام المزدوج بناءً على مستويات خطورة الحالة (Vulnerability Levels).

## 🚀 التقنيات المستخدمة (Technologies Used)
* **لغة البرمجة:** Java (JDK 21)
* **واجهة المستخدم:** JavaFX & Scene Builder
* **إدارة المشروع:** Maven
* **قاعدة البيانات:** MySQL / PostgreSQL
* **التنسيق الجمالي:** CSS Custom Styling

## 📐 النمط المعماري للمشروع (Architecture Pattern)
التطبيق يتبع نمط التصميم الشهير **MVC (Model-View-Controller)** لضمان فصل الكود وتنظيمه وسهولة صيانته:
* **Model:** يمثل كلاسات البيانات والحقول (Entities).
* **View:** ملفات الـ FXML والتصاميم المرئية لـ Dashboard والواجهات الفرعية.
* **Controller / DAO:** الكلاسات المسؤولة عن معالجة الأحداث والربط البرمجي مع قاعدة البيانات وإجراء عمليات الـ CRUD وفحص التكرار.

## 🔒 آلية منع التكرار الذكي (Duplicate Check Logic)
* يتم فحص رقم هوية العائلة والتاريخ الحالي تلقائياً.
* الحالات ذات مستوى الخطورة **HIGH** يُسمح لها بالاستلام نظراً لحاجتها الماسة.
* الحالات ذات مستوى الخطورة **MEDIUM / LOW** يتم رفضها تلقائياً مع إظهار تنبيه تحذيري للمنسق لمنع الهدر.
